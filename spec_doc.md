# MagicPod自習教材 仕様書

## 1. コンセプト

MagicPodの基本操作をステージクリア形式で自習できるWebベースの学習教材。  
Salesforce Trailheadを参考にした構成で、ステージを進めるごとにテストケースが育っていく体験を提供する。

---

## 2. 全体構成

### 編構成

| 編 | テーマ | ステージ数 |
|---|---|---|
| 入門編 | 基本コマンド（Web） | 8 |
| 中級編（将来） | 変数・条件分岐・繰り返し | 〜8 |

```
├── stage1-success.html # ステージ1 クリアページ

└── ...
```

---

## 3. 学習フロー

1. `index.html` で進め方を確認する
2. MagicPodにテストケースを1つ作成し、URLに変数 `stage_num` を設定する
3. `stage{{stage_num}}.html` を開き、更新履歴をヒントにテストステップを追記する
4. テストを実行し、`stageN-success.html` で「クリア！」テキストが検証できればクリア
5. 3回再実行して結果が安定していることを確認する
6. `stage_num` の値を次のステージ番号に更新し、次のステージへ進む

---

## 4. ページ仕様

### stageN.html（テスト対象ページ）

- ステージごとに改修が重なった「実在するWebページ」を模したUI
- ページ内に更新履歴メモを掲載し、これがテスト追記のヒントになる
- 更新履歴の形式は全ステージで統一する

```
更新履歴
v1.0  初回リリース：クリアボタンを設置
v1.1  クリアボタンの表示に3秒の遅延を追加
...
```

### stageN-success.html（クリアページ）

- デザイン・構成は全ステージで統一する
- 「クリア！」テキストを必ず含む（検証コマンドのターゲット）
- 「次のステージへ」ボタンを設置し、ユーザーが手動で次に進む

---

## 5. テストケース設計方針

- テストケースは1つを使い回す（ステージごとに削除せず追記していく）
- URLの `stage_num` 変数でステージを切り替える
- **全ステージで検証コマンドを含める**（操作するだけでなく確認する習慣を養う）
- 3回再実行して安定していることをクリア条件とする

## 6. 入門編 ステージ一覧

| ステージ | メインで学ぶこと | 検証コマンドの組み込み例 |
|---|---|---|
| 1 | タップ（クリック） | 「クリア！」テキストを検証 |
| 2 | 要素の待機 | ボタンが表示されたことを検証してからクリック |
| 3 | テキスト入力 | 入力後に表示されるテキストを検証 |
| 4 | セレクト／ラジオ | 選択結果の表示内容を検証 |
| 5 | スクロール | スクロール後に要素が見えていることを検証 |
| 6 | 複数ウィンドウ／タブ | 新タブのタイトルや内容を検証 |
| 7 | 共有ステップ | 共有ステップの呼び出し前後の状態を検証 |
| 8 | 総まとめ | 複合的な検証でテストの安定性を意識する |

## 7. デザイン仕様

### コンセプト
ゲームっぽくポップな宇宙テーマ。MagicPodのロケットアイコンの世界観と統一する。

### カラー
- メインカラー：`#24b857`（MagicPodブランドカラー）
- 学習ページ系の背景：ダーク宇宙（`#0d1b2a`）

### ページ別デザイン方針

| ページ | デザイン | 備考 |
|---|---|---|
| index.html | ダーク宇宙背景 + ステージグリッド | クリア済み／進行中／ロックを視覚的に表現 |
| stageN.html | 架空サービス「Galactic Survey System」風 | 惑星調査をテーマにしたWebアプリUI |
| stageN-success.html | ダーク宇宙背景で全ステージ統一 | 「クリア！」テキスト＋Tips＋次のステージへボタン |

### success.htmlの構成要素
- 「クリア！」テキスト（検証コマンドのターゲット）
- クリアしたステージ名
- 一言Tips（そのステージで学んだことに関連するアドバイス）
- 「次のステージへ」ボタン

## 8. 技術構成

| 項目 | 内容 |
|---|---|
| ホスティング | GitHub Pages（ベータ）|
| フロントエンド | HTML / CSS / JavaScript（フレームワークなし）|
| テストツール | MagicPod（Webアプリ） |
| テストケース管理 | ユーザー自身のMagicPodプロジェクト内 |
| 進捗データ保存 | localStorage |

### localStorage設計

```json
{
  "clearedStages": [1, 2],
  "currentStage": 3
}
```

- ブラウザのキャッシュ削除で進捗がリセットされる旨をindex.htmlに明記する
